Autorização
Para realizar requisições GET e POST nos endpoints da API você precisa de uma chave de autorização. Chamamos essa chave de token.

Para ter acesso ao token, é necessário que o usuário master da licença efetue a liberação deste pela interface do Feegow Clinic.

Para utilizar o token basta adicioná-lo ao header nas requisições com a chave x-access-token.

Header
Parâmetro	Token
x-access-token	eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9.eyJ1c2VyTA...
Em caso de dúvidas entre em contato com nosso suporte.

GET /appoints/available-schedule

Lista todos os horários disponíveis para uma especialidade ou procedimento.

Pode ser filtrado por unidade e profissional, possui um período de data para pesquisa.

Query Params
Parâmetro	Tipo	Descrição
tipo	numeric	E = Especialidade
P = Procedimento
especialidade_id
caso tipo = E	numeric	Identificação da especialidade
procedimento_id
caso tipo = P	numeric	Identificação do procedimento
data_start	date	Listar a partir dessa data
dd-mm-YYYY
data_end	date	Limitar consulta até essa data
dd-mm-YYYY
unidade_id
(opcional)	numeric	Identificação da unidade
Se unidade_id for igual a 0 será considerado a unidade principal, se não informado, será considerado todas as unidades
profissional_id
(opcional)	numeric	Identificação do profissional
convenio_id
(opcional)	numeric	Identificação do convênio
age_from
(opcional)	numeric	Idade mínima do paciente para atendimento com o profissional
age_to
(opcional)	numeric	Idade máxima do paciente para atendimento com o profissional
Respostas
Código	Tipo	Descrição
200	sucesso	Operação bem sucedida

Cria um novo agendamento.

 Não é permitido criar agendamentos retroativos. Data deve ser maior ou igual a data atual.

centavos

1 = Com convênio

Parâmetro	Tipo	Descrição

DATA_INICIO	date	dd/mm/YYYY

 Existe um limite de 60 dias entre o intervalo das datas para qualquer relatório.
